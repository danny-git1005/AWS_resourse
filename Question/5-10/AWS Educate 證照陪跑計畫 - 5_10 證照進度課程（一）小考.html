Geolocation Routing: This option is incorrect because, although it can reduce latency to some extent by geographical positioning, its primary purpose is to optimize geographical positioning, commonly used when content delivery needs to comply with regional restrictions or regulations. It is not specifically designed to minimize latency.

B. Latency Routing: This option is correct. Latency Routing policy optimizes user request routing based on the lowest network latency.

C. Weighted Routing: This is typically used for experiments or phased deployments. By setting different weights, it allows precise control over the proportion of traffic on new services. This strategy is mainly used when wanting to share resources with servers close geographically, providing more detailed control over geographical positioning. However, its primary purpose remains the optimization of geographical positioning, not minimizing latency.

4. Which of the following is a correct relationship between regions, Availability Zones, and edge locations?

 *
10/10
建議
Regions: These are separate geographic areas, each containing multiple Availability Zones. AWS divides the world into regions (e.g., us-east-1, eu-west-1), and each region represents a separate geographic area where AWS services are hosted.

Availability Zones: These are distinct locations within a region, each designed to be isolated from failures in other Availability Zones. They are interconnected with low-latency network links. Typically, a region consists of multiple Availability Zones (usually three or more), providing redundancy and fault tolerance.

Edge locations: These are endpoints for AWS's Content Delivery Network (CDN) service, Amazon CloudFront. They are located in various cities around the world and are used for caching content closer to end-users to improve performance. Edge locations are not contained within Availability Zones; rather, they are separate entities in the AWS infrastructure.
 
 
 
5. If an EC2 instance is deployed in a private subnet and needs to download content from the internet, which AWS service would allow this without exposing the instance to direct internet access?
 *
10/10
建議
A. Internet Gateway: This option is incorrect because Internet Gateways provide a way for instances to be reachable from the internet, which is not applicable for instances in a private subnet that should not be directly exposed to the internet.

B. NAT Gateway: This is the correct answer. A NAT Gateway enables instances in a private subnet to initiate outbound internet traffic (such as downloading files) without allowing inbound internet traffic, thereby keeping the instances private.

C. Elastic Load Balancer: While Elastic Load Balancers can route incoming internet traffic to instances, they do not facilitate outbound access from a private subnet to the internet for the purpose of downloading content.

D. AWS Direct Connect: This service primarily provides a private connection between an on-premises network and AWS but does not provide internet access to instances in a private subnet.

10. In the context of utilizing Amazon Web Services for application development, various interfaces facilitate interactions with AWS resources. Which of the following methods are valid for developers to communicate with AWS? (Choose three.) *
10/10
必填
建議
A. AWS Management Console
The AWS Management Console is a web-based interface that provides a user-friendly dashboard from which users can access and manage AWS services. This console allows for visual interaction with AWS resources, making it easy for users to create, configure, and monitor their AWS deployments.

B. AWS Command Line Interface (CLI)
The AWS CLI is a unified tool that provides a command-line interface to interact with AWS services. This tool allows users to perform many of the same actions available in the Management Console but through command line scripts, making it suitable for automation and repetitive tasks.

C. AWS Software Development Kits (SDKs)
AWS SDKs are available for various programming languages such as Python, Java, Ruby, .NET, etc. These kits enable developers to interact with AWS services programmatically, allowing for the integration of AWS features directly into applications.
